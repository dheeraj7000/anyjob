the pipeline

Five steps, in this exact order

Each stage hands off to the next — nothing here runs out of sequence, and the chain always ends with you, not a click on your behalf.

01

Log in for real

Playwright drives your actual username/password through the site's own login form. No session cookies stored, no scraped tokens — npm run login once per site.

02

Crawl the postings

Adapters for Workday and Greenhouse boards pull listings and full descriptions into a flat local store — npm run crawl.

03

Parse your resume once

Upload a PDF/DOCX/TXT and an LLM lifts it into a structured profile — your real name, history, and skills, cached at data/profile.json.

04

Map fields, not guesses

Every labeled input on the form is scanned and matched against your profile. Anything the model can't confidently answer is left blank and flagged, never fabricated.

05

Screenshot, then stop

The browser window stays open, filled fields are highlighted, and the run ends. You read it over and click Submit yourself.

two ways in

Pick the browser you trust more

Same field-mapping engine underneath — the difference is whose browser session does the filling.

Playwright CLI

A browser anyjob drives

  • Runs your real login through npm run login, session cached locally
  • Best for sites behind Workday-style credentialed auth
  • Opens a visible (non-headless) Chromium window to review before you submit
  • npm run apply -- workday:asu <jobId>

Chrome side panel

Your own already-logged-in tab

  • No credentials touch anyjob at all — it never logs in for you
  • Docks open in the side panel across tab navigation, one click: Scan & Fill
  • Attaches your uploaded resume/cover letter to file-upload fields directly
  • Talks to a local server on 127.0.0.1, gated by a token only you have

the extension

Upload once, fill everywhere

  • Upload your resume and the panel extracts your real name, contact info, and history — no more placeholder data in your forms.
  • Upload a cover letter once and it's attached automatically to any cover-letter file field it finds.
  • Every fill is visible — filled fields outline green, anything skipped outlines red with the reason why, right on the page.

quickstart

Running locally

Everything stays on your machine — credentials, resume, and the LLM calls you configure yourself.

# install & configure
$ npm install
$ npx playwright install chromium
$ cp .env.example .env

# parse your resume once, log in once per site
$ npm run parse-resume -- ./data/resume.pdf
$ npm run login -- workday:asu

# or: start the local server and use the browser extension instead
$ npm run start-all
